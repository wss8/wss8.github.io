1、值类别

1.1、右值引用的引入

引用的本质

  • 编译器隐藏了引用变量内部保存引用对象的地址的事实
  • 引用变量也可以使用const修饰：const引用 和 non-const引用
  • const引用代表编译器层面禁止修改引用变量的值（不能再直接出现在赋值符号左边），但依然可以间接修改

左值引用（int&）

  • 左值引用可以引用一个lvalue（int& r = i）
  • non-const左值引用不能引用一个prvalue(纯右值)，const左值引用可以绑定到一个prvalue(如：const int& r = 10; const Test& = Test();)

右值引用(int&&)

  • 右值引用使得non-const引用变量能够引用prvalue（int&& r = 10），新特性解决了C++98/03的不足，且兼容
  • 反汇编发现：右值引用和const左值引用生成的汇编代码完全一致
  • 右值引用不能用来引用一个lvalue
    • 可以使用 static_cast<int&&>(lvalue) 将lvalue转为xvalue，这样就可以引用了
    • std::move() 封装了 上述的强制类型转换过程，并赋予了新的语义（即期望临时对象资源被转移，本身不提供转移资源功能）
  • 临时对象的生命只有一行，右值引用能够有效的利用临时对象持有的系统资源，实现资源转移，避免不必要的大量复制
    • 函数传参可能传入临时对象，在C++98/03中会使用const Test& 接收，虽可以自主选择拷贝资源或转移资源，但倾向于深拷贝资源
    • C++11在兼容过去的同时，可以使用右值引用接收实参，倾向于自己将要转移资源，而不是拷贝（虽然也可以，但不这样干）
  • 注意：转移资源后应该断开临时对象释放资源的句柄，临时对象才可以安全析构

1.2、值类别（value categories）

明确表达式概念

  • 表达式是_运算符_及其_操作数_的序列，语句是指一条语句，以分号作为结束标志
  • 因此一条语句可能不止一个表达式。如：int a = func(100); a = (a > b); a = ((a > b ? a : b) == a) ? a : b;
  • 构造函数的直接调用创建临时对象，属于函数调用，因此也是表达式。T var = T(); 由两个表达式组成，T()是子表达式

每个C++表达式都有两个独立的属性: 类型 （type）和 值类别（value categories）

  • 类型：用来表示表达式结果类型的属性
    • 可以使用decltype推测表达式结果值的类型。如：decltype((x)) 、decltype(x = 10)
  • 值类别：用来描述表达式结果值的一个属性，通过表达式的值对表达式进行分类
    • 在最早的C中，lvalue是指 赋值符号左边的操作数，后来C中的 lvalue 代表locator value，用作定位标识对象
    • C++11之前只有两类，lvalue 和 rvalue （概念不够明确且有争议）
    • C++11引入移动语义的引入，值类别被重新定义，引入了 prvalue（纯右值）, xvalue（将亡值）, glvalue（泛左值）的概念
      • prvalue 和 xvalue 可称为 rvalue
      • lvalue 和 xvalue 可称为 glvalue
      • 每个表达式的值类别可以是 lvalue，xvalue 或 prvalue中的一个，或者说是 rvalue、glvalue中的一个
    • 定义
      • have identity and cannot be moved from are called lvalue expressions;
      • have identity and can be moved from are called xvalue expressions;
      • do not have identity and can be moved from are called prvalue (“pure rvalue”) expressions;
      • 个人理解的 have identity：程序中有对象的名字，指针或引用（直接表现出来的，而不是通过重载间接获得）
        • 如：const int& i = 10；// 10是prvalue， 中间会被转换为xvalue，被 i 引用
    • rvalue特性
      • 内置地址运算符不能获取rvalue的地址。如：&10； &Test(); &(Test().a); 都是错误的（Test类重载&可以返回this，但没用）
      • rvalue可用于初始化const左值引用，在这种情况下，由rvalue标识的对象的生存期将延长，直到引用范围终止
      • rvalue不能用作内置赋值或复合赋值运算符的左侧操作数。如：Test().a = 10; 10+=10; 都是错误的，注意Test() = Test()调用的类中重载的赋值操作符
      • rvalue可用于初始化右值引用，在这种情况下，由rvalue标识的对象的生存期将延长，直到引用范围终止
      • 当用作函数参数且有两个重载的函数（ 一个右值引用参数和一个const左值引用参数）时, 右值将绑定到右值引用的哪个函数（因此，如果存在移动构造和拷贝构造，一个rvalue参数调用移动构造函数）
#include <iostream>
using namespace std;

int g = 20;
int& func() { return g; }
int&& xvalue() { return static_cast<int&&>(g); } 

class Test 
{ 
public:
    int a;
    //Test* operator & (){ return this; }
};

int main()
{
    // 猜测：lvalue表达式的结果为lvalue，所以下面语句a是变量，再后来a又是lvalue
    int a = 10;     // lvalue表达式 ，10是prvalue，a是变量。
    bool f = false; // lvalue表达式，f是变量，false是prvalue
    int& l = a;     // lvalue表达式， l是左值引用变量，a是lvalue
    int&& r = 10;   // lvalue表达式 ，r是右值引用变量，10是prvalue
    func();         // lvalue表达式
    int b = func(); // lvalue表达式 ，b是变量, func()也是lvalue表达式且表达式类型是int&
    b = static_cast<int&>(a); // lvalue表达式, static_cast<int&>(a)也是一个lvalue表达式
    a = r;   // lvalue表达式，r是lvalue, a是lvalue
    a += r;  // lvalue表达式
    ++a;     // lvalue表达式
    int c = (a, r); // lvalue表达式 ，(a, r)是一个lvalue表达式,其中r是一个lvalue
    "Hello";        // 这也是一个特殊的lvalue
    char arr[] = "World"; // "World"是lvalue, arr[i]是变量
    arr[0] = 'a';   // 'a'是prvalue，可修改的原因是存储于只读存储区的字符串被拷贝一份到栈内

    int d = (a < b);  // lvalue表达式，(a < b)是一个prvalue表达式，(a < b)子表达式产生的结果为一个布尔值

    // 下面一行所有的表达式产生的结果都是prvalue, 他们都是prvalue表达式
    a++; a--; a + b; a % b; a & b; a << b; a && b; a || b; !a; a < b; a == b; a >= b; &a;

    Test t = Test(); // lvalue表达式，Test()是prvalue表达式

    t = std::move(Test());     // lvalue表达式。Test()是prvalue表达式，std::move(Test())是xvalue表达式
    d = xvalue();              // lvalue表达式，xvalue()是一个xvalue表达式且表达式类型为int&&
    d = static_cast<int&&>(g); // lvalue表达式，static_cast<int&&>(g)是xvalue表达式
    Test().a;                  // 整体是lvalue表达式，Test()是一个rvalue, Test().a是一个xvalue表达式

    return 0;
}

1.3、值类别的转换

int main()
{
    int a = 10;    
    bool f = false;

    // 1. lvalue->prvalue
    a = f;   
    // 实际是语句会被分为两条指令
    // movzx  eax, byte ptr [f]  
    // mov    dword ptr[a], eax
    // 发现这里发生了转换（lvalue->prvalue 或 gvalue->prvalue）

    // 2. prvalue->xvalue
    const int& ci = 10;// 本质为将10放到一个临时局部变量里，引用变量引用局部变量，编译器层面禁止直接修改这个隐藏的局部变量,可以修改这个局部变量: const_cast<int&>(ci) = 20
    //const Test& tt = Test(10);
    // 实际是语句会被分为三条指令
    // mov  dword ptr [ebp-54h],0Ah  
    // lea  eax, [ebp - 54h]
    // mov  dword ptr[ci], eax
    // 存在引用，引用了这个临时变量，所以有identify
    // 发生了prvalue->xvalue的转化
    int&& rr = 10;
    // mov  dword ptr[ebp - 3Ch], 0Ah
    // lea  eax, [ebp - 3Ch]
    // mov  dword ptr[rr], eax
    // 发现和const int& ci = 10;的汇编语句几乎一致，只是编译时ci不能再作为左操作数
    // 也发生了prvalue->xvalue的转化
    rr = 20; // 如果使用const右值引用，使得编译器层面禁止修改引用变量

    // 3. lvalue->xvalue
    int& l_ref = static_cast<int&>(a);
    // lea  eax, [a]
    // mov  dword ptr[l_ref], eax
    int&& r_ref = static_cast<int&&>(a);
    // lea  eax, [a]
    // mov  dword ptr[r_ref], eax
    // 右值引用本就和移动语义息息相关，使用右值引用应该就是想move，如果不move, 用const左值引用不香吗？
    // static_cast<int&&>(a);是xvalue表达式
    // static_cast<int&>(a);是lvlaue表达式

    return 0;
}

2、右值引用、移动构造与移动赋值

右值引用实现资源转移

#include <iostream>
using namespace std;

class Test
{
    int* m_pvalue = nullptr; // 指向系统的堆资源
public:
    Test(int val = 0) // explicit
    {
        this->m_pvalue = new int(val);
    }
    Test(const Test& obj)
    {
        this->m_pvalue = new int(*const_cast<Test&>(obj));
    }
    int operator * () const
    {
        return m_pvalue != nullptr ? *m_pvalue : -1;
    }
    Test& operator = (const Test& obj)
    {
        if (this != &obj)
        {
            delete this->m_pvalue;
            this->m_pvalue = new int(*const_cast<Test&>(obj));
        }

        return *this;
    }
    ~Test() {
        delete m_pvalue;
        m_pvalue = nullptr;
    }

    void copy(const Test& obj)
    {
        delete this->m_pvalue;
        this->m_pvalue = new int(*obj); // 资源的拷贝
    }
    void move(const Test& obj)
    {
        delete this->m_pvalue;
        this->m_pvalue = obj.m_pvalue;  // 资源的转移
        obj.m_pvalue = nullptr;
    }
    void move(Test&& obj)
    {
        cout << "void move(Test&& obj)" << endl;
        delete this->m_pvalue;
        this->m_pvalue = obj.m_pvalue;  // 资源的转移
        obj.m_pvalue = nullptr;
    }
};

int main()
{
    Test op;
    op.copy(Test(100)); 
    op.move(Test(200));

    Test tmp(300);
    op.move(std::move(tmp)); 
    // std::move(tmp)为xvalue表达式，当rvalue用作函数参数且有两个重载的函数（ 一个右值引用参数和一个const左值引用参数）时
    // 右值将绑定到右值引用的那个函数（因此，如果存在移动构造和拷贝构造，rvalue参数调用移动构造函数）
    cout << *tmp << " == -1 说明资源被转移了" << endl;

    return 0;
}

C++11引入了移动构造函数与赋值构造函数

  • 拷贝构造函数可以接收 lvalue 和 rvalue，但它实现的本义倾向于资源的深拷贝
  • 移动构造函数参数为右值引用，可以是const或非const右值引用，本义倾向于资源转移，一个类可以具有多个移动构造函数
  • https://en.cppreference.com/w/cpp/language/move_constructor
#include <iostream>
using namespace std;

class Test
{
    int* m_pvalue = nullptr;
public:
    Test(int val = 0) // explicit
    {
        cout << "Constructor " << val << endl;
        this->m_pvalue = new int(val);
    }
    Test(const Test& obj)
    {
        cout << "Copy Constructor" << endl;
        this->m_pvalue = new int(*const_cast<Test&>(obj));
    }
    int operator * () const
    {
        return m_pvalue != nullptr ? *m_pvalue : -1;
    }
    Test& operator = (const Test& obj)
    {
        cout << "Copy Assignment operator" << endl;
        if (this != &obj)
        {
            delete this->m_pvalue;
            this->m_pvalue = new int(*const_cast<Test&>(obj));
        }

        return *this;
    }
    ~Test() {
        cout << "~Test() : " << *(*this) << endl;
        delete m_pvalue;
        m_pvalue = nullptr;
    }
    // 移动构造
    Test(Test&& obj) : m_pvalue(obj.m_pvalue)
    {
        cout << "Move Constructor" << endl;
        obj.m_pvalue = nullptr;
    }
    // 移动赋值
    Test& operator = (const Test&& obj)
    {
        if (this != &obj)
        {
            cout << "Move Assignment operator" << endl;
            delete this->m_pvalue;
            this->m_pvalue = const_cast<Test&&>(obj).m_pvalue;
            const_cast<Test&>(obj).m_pvalue = nullptr;
        }

        return *this;
    }
};

int main()
{
    Test t0(50);             // Constructor
    Test t1 = Test(100);     // Constructor + Move Constructor + ~Test() : -1 rvalue优先匹配移动构造(右值引用 )
    Test t2 = std::move(t0); // Move Constructor
    cout << *t0 << endl;    // -1
    cout << *t1 << endl;    // 100
    cout << *t2 << endl;    // 50

    t0 = t1;                 // Copy Assignment operator
    cout << *t0 << " " << *t1 << endl;  // 100 100
    t0 = std::move(t1);      // Move Assignment operator
    cout << *t0 << " " << *t1 << endl;  // 100 -1

    string s1 = "wss";
    cout << s1.c_str() << "\t" << (void*)(s1.c_str()) << endl; // wss   0xbfbcf4ac
    string s2 = std::move(s1); // 调用basic_string(basic_string&& __str)窃取资源

    cout << s1.c_str() << "\t" << (void*)(s1.c_str()) << endl; // ""    0xbfbcf4ac
    cout << s2.c_str() << "\t" << (void*)(s2.c_str()) << endl; // wss    0xbfbcf4dc  
    // 发现移动构造后字符串的地址发生了改变，说明string的移动构造还是会执行深拷贝，效率不高

    string&& s3 = std::move(s2);
    s3 = "hello";
    cout << s2.c_str() << "\t" << (void*)(s2.c_str()) << endl; // hello    0xbfbcf4dc
    cout << s3.c_str() << "\t" << (void*)(s3.c_str()) << endl; // hello    0xbfbcf4dc

    return 0;
}

3、返回值优化(RVO)

  • RVO：返回值优化（Return Value Optimization），C++98已经支持的特性，是一个编译优化技术
    • 函数返回时不调用拷贝构造或移动构造操作生成临时对象（省去了函数返回时的拷贝构造或移动构造）
    • 初始化对象时不创建只用于初始化相同类型的另一个对象的临时对象
    • g++编译器指定 -fno-elide-constructors 选项可以禁用该优化
  • NRVO: 命名返回值优化（Named Return Value Optimization），RVO的升级
    • RVO是对匿名对象（临时对象）的优化，NRVO可以支持有名对象优化
  • 后文不严格区分两者，只当作是一种RVO优化技术
#include <iostream>
using namespace std;

class Test
{
    int* m_pvalue = nullptr;
public:
    Test(int val = 0) // explicit
    {
        cout << "Constructor " << hex << val << endl;
        this->m_pvalue = new int(val);
    }
    Test(const Test& obj)
    {
        cout << "Copy Constructor" << endl;
        this->m_pvalue = new int(*const_cast<Test&>(obj));
    }
    int operator * () const
    {
        return m_pvalue != nullptr ? *m_pvalue : -1;
    }
    Test& operator = (const Test& obj)
    {
        cout << "Copy Assignment operator" << endl;
        if (this != &obj)
        {
            delete this->m_pvalue;
            this->m_pvalue = new int(*const_cast<Test&>(obj));
        }

        return *this;
    }
    ~Test() {
        cout << "~Test() : " << hex << *(*this) << endl;
        delete m_pvalue;
        m_pvalue = nullptr;
    }
    // 移动构造
    Test(Test&& obj) : m_pvalue(obj.m_pvalue)
    {
        cout << "Move Constructor" << endl;
        obj.m_pvalue = nullptr;
    }
    // 移动赋值
    Test& operator = (const Test&& obj)
    {
        if (this != &obj)
        {
            cout << "Move Assignment operator" << endl;
            delete this->m_pvalue;
            this->m_pvalue = const_cast<Test&&>(obj).m_pvalue;
            const_cast<Test&>(obj).m_pvalue = nullptr;
        }
        return *this;
    }
};

Test getA()
{
    cout << "getA()" << endl;
    return Test(0xAAA); // C++98/03 关闭RVO时：生成临时对象Test(0xAAA)，函数返回，调用拷贝构造再次生成临时对象Test(0xAAA)，供调用者使用
}
Test getB()
{
    cout << "getB()" << endl;
    Test tmp(0xBBB); // 有名对象

    return tmp; // C++98/03 关闭RVO：函数返回时调用拷贝构造生成临时对象，析构tmp
}

Test&& getC()
{
    cout << "getC()" << endl;
    Test tmp(0xCCC); // 生成局部对象
    return std::move(tmp); 
    // 由于返回值类型为右值引用，所以不会再调用拷贝构造或移动构造
    // 这种错误和返回局部对象的左值引用或局部对象的指针原理一样。函数执行完栈帧销毁，局部对象失效
}
int main()
{
    /* vs2015执行过程(默认开启了RVO) */

    Test t1(8); // 为t1分配内存（call Test::__autoclassinit2），再调用构造函数初始化
    Test t2 = 8;
    Test t3 = Test(8);
    Test t4(Test(8));
    // 开启RVO时，上方四条语句都仅调用一次普通构造，生成的汇编代码完全一致，说明没有生成Test(8)临时对象
    // 关闭RVO时，t2,t3,t4调用了 普通构造 + 拷贝构造/移动构造 + 析构函数，说明生成了Test(8)临时对象
    // 当编译选项使用c++17时，是否开启RVO, 都只调用普通构造

    cout << endl << endl;

    Test t5 = getA(); // 编译器行为：为t5分配内存，将t5内存地址传入eax寄存器，eax入栈，在A函数中调用构造函数进行初始化t5
// 使用函数返回值初始化结构体变量时：用eax存变量地址，eax入栈，返回地址入栈，调用函数，新函数将变量地址（ebp+8）赋给ecx，再调用构造函数
    Test t6(getA());  
    // 开启RVO时, 上方两条语句都只调用一次普通构造，说明没有生成临时对象
    // 关闭RVO时，都调用了 普通构造 + (移动构造+移动构造) / (拷贝构造+拷贝构造)

    cout << endl << endl;

    Test t7 = getB();  // NRVO
    // 为t7分配内存，将t7地址传入eax寄存器, eax入栈，调用B函数，B函数中为tmp申请内存，调用构造函数初始化tmp,
    // 再通过t7地址（ebp+8赋给ecx）调用移动构造函数，进行资源转移（或拷贝构造进行资源拷贝）
    Test t8(getB());
    // 开启RVO时, 上方两条语句都调用一次普通构造 + 移动构造/拷贝构造
    //（gcc下只会调用一次普通构造, 使用objdump-S反汇编发现函数B在前言部分将t8地址(ebp+8)赋给了局部变量，局部变量又赋给ecx,调用构造函数初始化tmp其实初始化的是ecx指向的t8）
    // 关闭RVO时，都调用了 普通构造 + (移动构造+移动构造) / (拷贝构造+拷贝构造)

    cout << endl << endl;

    t7 = getA();// 对于这种不是初始化的操作，必须得执行赋值操作，为啥？因为赋值操作函数可能需要执行一些其他操作(如释放资源等），肯定不能优化掉
    // 编译器会选择一个局部变量的地址x入栈，A函数在x初始化对象, 通过eax返回对象地址，ecx存放t7地址，调用移动赋值操作
    t5 = getB(); // 和上面做法类似
    // 开启RVO时, t7所在行调用了普通构造 + 移动赋值/拷贝赋值， t8所在行调用了普通构造 + 移动构造/拷贝构造 + 移动赋值/拷贝赋值（gcc依旧会少了一个，原理前面解释过了）
    // 关闭RVO时，都调用了 普通构造 + 移动构造/拷贝构造 + 移动赋值/拷贝赋值

    //cout << endl << endl;

    //t5 = getC();
    //cout << *t5 << endl; // error 段错误或垃圾数据

    return 0;
}

4、万能引用与引用折叠

模板函数传参时编译器推导时可能遇到下面四种特殊情况

  • 左值引用 + 右值引用 int& && -> int&
  • 左值引用 + 左值引用 int& & -> int&
  • 右值引用 + 左值引用 int&& & -> int&
  • 右值引用 + 右值引用 int&& && -> int&&
  • 编译器采用引用折叠，规则： 如果有左值引用那么就折叠为左值引用，否者就是右值引用
#include <iostream>
#include <boost/type_index.hpp>
using namespace std;

using boost::typeindex::type_id_with_cvr;  // Effective Modern C++有介绍：可以查看编译器的模板推导类型结果

template<typename T>
void func(T&& parm) // 这样写就是万能引用：参数传递lvalue就是左值引用,传递rvalue就是右值引用
{
    cout << "T: " << type_id_with_cvr<T>().pretty_name() << endl;
    cout << "parm: " << type_id_with_cvr<decltype(parm)>().pretty_name() << endl;
}

template<typename T, unsigned int N>
void f1(T(&arr)[N])
{
    cout << N << endl;
    cout << "T: " << type_id_with_cvr<T>().pretty_name() << endl;
    cout << "arr: " << type_id_with_cvr<decltype(arr)>().pretty_name() << endl;
}

int main()
{
    int lvalue = 10;            
    const int& l_ref = lvalue;  
    char arr[] = "wss";

    func(10);     // ==> func<int>(10); -> int&&  实参是rvalue所以形参是右值引用
    cout << endl; 

    func(lvalue); // ==> func<int&>(lvalue); -> int& &&  ->引用折叠 -> int&   实参是lvalue，所以形参是左值引用
    cout << endl;

    // 参数为rvalue时, 组合为 int&& ; lvalue时 T被推导为int&, 组合为int& && -> int&
    // 实参左rvalue, 形参就为右值引用，实参为lvaue时形参就为左值引用
    // 也就是说万能引用够判断原始实参为lvalue还是rvalue，是实现完美转发的关键

    func(l_ref);  // func<const int&>(l_ref);  -> const int&   const左值引用
    cout << endl;

    f1(arr);     // f1<char, 4>() -> char (&)[4]  左值引用
    cout << endl;
    func(arr);   // func<char(&)[4]>(arr);   左值引用

    return 0;
}

5、完美转发

#include <iostream>
using namespace std;

void func(int&& t1, int& t2)
{
    cout << "void func(int&& t1, int& t2)" << endl;
    t2++;
}

template <typename F, typename T1, typename T2>
void test(F func, T1&& t1, T2&& t2)
{
    // func(t1, t2);  
    // 直接调用上面函数会报错，不完美
    // 原因：t1为右值引用，实参为rvalue时，完全合理，但t1为一个lvalue，lvalue作为实参传给func右值引用形参,出错
    // 需要的特性：原来是rvalue,再次传参还是rvalue，lvalue也是
    // 解决：把lvalue转变为xvalue，之前就提过可以使用static_cast, move
    func(static_cast<T1&&>(t1), static_cast<T2&&>(t2)); 
    func(std::move(t1), t2);

    // 标准库提供std::forward来保持左值或右值
    func(std::forward<T1>(t1), std::forward<T2>(t2));
}

int main()
{
    int k = 1;

    test<void(int&&, int&)>(func, 0, k); 
    // 0是rvalue ，k是lvalue ，编译器推导T1 = int, T2 = int&, F已显式指定为void(int&&, int&)

    cout << k << endl;

    int&& r1 = std::forward<int>(k);          
    //int&& r2 = std::forward<int&>(k); // error int& && -> int& 所以r2需要为int&类型

    return 0;
}

/// remove_reference
template<typename _Tp>
struct remove_reference { typedef _Tp   type; };
template<typename _Tp> 
struct remove_reference<_Tp&> { typedef _Tp   type; };
template<typename _Tp>
struct remove_reference<_Tp&&> { typedef _Tp   type; };

// 发现std::remove_reference<_Tp>的作用是删除引用类型，并通过type别名返回无引用的类
/**
   *  @brief  Convert a value to an rvalue.
   *  @param  __t  A thing of arbitrary type.
   *  @return The parameter cast to an rvalue-reference to allow moving it.
  */
template<typename _Tp>
constexpr typename std::remove_reference<_Tp>::type&& move(_Tp&& __t) noexcept
{ 
      return static_cast<typename std::remove_reference<_Tp>::type&&>(__t); 
}

/**
   *  @brief  Forward an lvalue.
   *  @return The parameter cast to the specified type.
   *
   *  This function is used to implement "perfect forwarding".
   */
template<typename _Tp>
constexpr _Tp&& forward(typename std::remove_reference<_Tp>::type& __t) noexcept
{ 
    return static_cast<_Tp&&>(__t); 
}
